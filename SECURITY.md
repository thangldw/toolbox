# Security Policy / Chính sách bảo mật / セキュリティポリシー

## Tiếng Việt

### Phiên bản được hỗ trợ

Chỉ phiên bản mới nhất của mỗi ứng dụng được nhận bản vá bảo mật.

### Báo cáo lỗ hổng

Không đăng công khai đường dẫn cá nhân, log chứa dữ liệu riêng tư, proof-of-concept có khả năng xóa dữ liệu, snapshot Changeora chưa làm sạch hoặc thông tin nhận dạng.

Hãy sử dụng **GitHub Security Advisories → Report a vulnerability** của repository. Nếu tính năng này chưa được bật, chỉ mở issue với mô tả tổng quát và yêu cầu kênh liên hệ riêng.

Báo cáo nên bao gồm phiên bản, macOS, điều kiện tái hiện, phạm vi dữ liệu bị ảnh hưởng và khả năng khôi phục. Không gửi dữ liệu thật nếu một fixture tối giản có thể tái hiện vấn đề.

<a id="en"></a>

## English

### Supported versions

Only the latest version of each application receives security fixes.

### Reporting a vulnerability

Do not publicly post personal paths, logs containing private data, proof-of-concepts capable of deleting data, unsanitized Changeora snapshots, or identifying information.

Use **GitHub Security Advisories → Report a vulnerability** for this repository. If private reporting is not enabled, open an issue containing only a general description and request a private contact channel.

Include the application version, macOS version, reproduction conditions, affected data scope, and recovery status. Do not submit real data when a minimal fixture can reproduce the issue.

## 日本語

### サポート対象バージョン

各アプリケーションの最新バージョンのみ、セキュリティ修正の対象です。

### 脆弱性の報告

個人のパス、非公開データを含むログ、データ削除が可能な proof-of-concept、未サニタイズの Changeora snapshot、個人を特定できる情報を公開しないでください。

この repository の **GitHub Security Advisories → Report a vulnerability** を使用してください。非公開報告が有効でない場合は、概要のみを記載した Issue を作成し、非公開の連絡方法を依頼してください。

アプリケーションの version、macOS version、再現条件、影響を受けるデータの範囲、復元可能性を含めてください。最小限の fixture で再現できる場合は、実データを送信しないでください。
